  • 連線設定：位於左側最上方，測試連線按鈕可快速檢查 SSH。
  • 全域控制：連線設定下方有「每次下指令清除舊訊息」勾選與綠色「使用說明」。

📘 linux.txt 說明與常見指令解釋

常用 LINUX 指令由 COMMANDS/linux.txt 載入，一行一筆，格式：

名稱 = 實際指令

例如：

列出 /mnt/usr/ 所有檔案 = ls -la /mnt/usr/

常見項目與用途

名稱（下拉顯示）實際指令用途/說明注意事項
列出 /mnt/usr/ 所有檔案 ls -la /mnt/usr/ 檢視 DUT 使用者資料夾內檔案清單，含權限/大小/時間。 大量輸出可配合右上搜尋功能定位檔名。
刪除 /mnt/usr/ JPG 檔 rm -f /mnt/usr/*.jpg 清理 JPG 圖片。 不可復原，請先用 ls 確認目錄與檔案。

刪除 /var/vsp/ JPG 檔 rm -f /var/vsp/*.jpg 清理 VSP 目錄下 JPG。 確認權限與路徑是否正確。
刪除 /var/vsp/ YUV 檔 rm -f /var/vsp/*.yuv 清理 VSP 目錄下 YUV。 確認權限與路徑是否正確。

管理建議

  • 可直接以文字編輯器維護 COMMANDS/linux.txt，新增或刪除專案需要的指令。
  • 下拉顯示會自動加上 1/2/3… 編號，僅為顯示用，不影響實際送出的指令。

📋 工具概述

4CAM_DEBUG_TOOL 是一個專為 4 攝影機 DUT（Device Under Test）設計的 SSH 控制工具，提供直觀的圖形介面來執行遠端指令和檔案傳輸。

🔧 主要功能

  • SSH 連線管理：自動連線到 DUT 設備
  • 指令執行：支援預設指令和常用 Linux 指令
  • 檔案傳輸：使用 SCP 從 DUT 下載檔案到 PC
  • 設定保存：自動保存和載入使用者設定
  • 字體調整：可調整介面字體大小

🚀 快速開始

  1. 啟動程式：執行 python main.py 或雙擊 EXE 檔案
  2. 檢查連線：程式會自動嘗試連線到 DUT
  3. 執行指令：選擇指令並點擊「執行指令」
  4. 檔案傳輸：設定來源和目標路徑，點擊「開始傳輸」

📁 檔案傳輸

支援的來源格式：

  • 單一檔案：/mnt/usr/image.jpg
  • Glob 模式：/mnt/usr/*.jpg
  • 多個檔案：/mnt/usr/test_*.log

目標目錄：

  • 支援 Windows 路徑格式：D:\VALO360CAM
  • 如果目錄不存在，會自動建立
  • 已存在的檔案會被覆蓋
💡 提示：檔案傳輸使用 SCP 協議，確保網路連線穩定。

⌨️ 指令操作

預設指令（Command.txt）：

  • 從 REF/Command.txt 載入預設指令
  • 支援下拉選單選擇
  • 指令格式：指令名稱 = 完整指令

常用 Linux 指令：

  • ls -la：列出檔案詳細資訊
  • ps aux：顯示執行中的程序
  • df -h：顯示磁碟使用量
  • free -h：顯示記憶體使用量
  • top：即時系統監控
  • netstat -an：顯示網路連線

🎛️ 介面操作

左側控制面板：

  • 連線設定：設定 DUT 和 PC 的 IP 位址
  • 指令控制：載入和執行預設指令
  • 常用指令：快速執行 Linux 指令
  • 檔案傳輸：設定來源和目標路徑

右側輸出面板：

  • 顯示指令執行結果
  • 顯示檔案傳輸狀態
  • 顯示錯誤訊息和除錯資訊
  • 支援「清空輸出」功能

🔍 故障排除

常見問題：

  1. SSH 連線失敗：
    • 檢查 DUT IP 位址是否正確
    • 確認 DUT 已開機且支援 SSH
    • 檢查網路連線
  2. 指令執行失敗：
    • 確認指令格式正確
    • 檢查 DUT 環境變數
    • 某些指令需要特定環境設定
  3. 檔案傳輸失敗：
    • 檢查來源路徑是否存在
    • 確認目標目錄權限
    • 檢查磁碟空間
⚠️ 注意事項：
  • 請確保 DUT 設備已正確設定網路連線
  • 某些 DUT 專用指令（如 diag）需要特定的環境變數
  • 檔案傳輸時請確保目標磁碟有足夠空間
  • 建議在執行重要操作前先測試連線
