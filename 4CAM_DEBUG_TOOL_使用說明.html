4CAM_DEBUG_TOOL 使用說明

📋 工具概述

4CAM_DEBUG_TOOL 是一個專為 4 攝影機 DUT（Device Under Test）設計的 SSH 控制工具，提供直觀的圖形介面來執行遠端指令和檔案傳輸。

🔧 主要功能

  • SSH 連線管理：自動連線到 DUT 設備
  • 指令執行：支援預設指令和常用 Linux 指令
  • 檔案傳輸：使用 SCP 從 DUT 下載檔案到 PC
  • 設定保存：自動保存和載入使用者設定
  • 字體調整：可調整介面字體大小

🚀 快速開始

  1. 啟動程式：執行 python main.py 或雙擊 EXE 檔案
  2. 檢查連線：程式會自動嘗試連線到 DUT
  3. 執行指令：選擇指令並點擊「執行指令」
  4. 檔案傳輸：設定來源和目標路徑，點擊「開始傳輸」

⚙️ 連線設定

參數預設值說明
DUT IP192.168.11.143DUT 設備的 IP 位址
PC IP192.168.11.142本機 PC 的 IP 位址
使用者名稱rootSSH 登入帳號
密碼（空）DUT 不需要密碼
超時時間30 秒指令執行超時時間

📁 檔案傳輸

支援的來源格式：

  • 單一檔案：/mnt/usr/image.jpg
  • Glob 模式：/mnt/usr/*.jpg
  • 多個檔案：/mnt/usr/test_*.log

目標目錄：

  • 支援 Windows 路徑格式：D:\VALO360CAM
  • 如果目錄不存在，會自動建立
  • 已存在的檔案會被覆蓋
💡 提示：檔案傳輸使用 SCP 協議，確保網路連線穩定。

⌨️ 指令操作

預設指令（Command.txt）：

  • 從 REF/Command.txt 載入預設指令
  • 支援下拉選單選擇
  • 指令格式：指令名稱 = 完整指令

常用 Linux 指令：

  • ls -la：列出檔案詳細資訊
  • ps aux：顯示執行中的程序
  • df -h：顯示磁碟使用量
  • free -h：顯示記憶體使用量
  • top：即時系統監控
  • netstat -an：顯示網路連線

🎛️ 介面操作

左側控制面板：

  • 連線設定：設定 DUT 和 PC 的 IP 位址
  • 指令控制：載入和執行預設指令
  • 常用指令：快速執行 Linux 指令
  • 檔案傳輸：設定來源和目標路徑

右側輸出面板：

  • 顯示指令執行結果
  • 顯示檔案傳輸狀態
  • 顯示錯誤訊息和除錯資訊
  • 支援「清空輸出」功能

🔍 故障排除

常見問題：

  1. SSH 連線失敗：
    • 檢查 DUT IP 位址是否正確
    • 確認 DUT 已開機且支援 SSH
    • 檢查網路連線
  2. 指令執行失敗：
    • 確認指令格式正確
    • 檢查 DUT 環境變數
    • 某些指令需要特定環境設定
  3. 檔案傳輸失敗：
    • 檢查來源路徑是否存在
    • 確認目標目錄權限
    • 檢查磁碟空間
⚠️ 注意事項：
  • 請確保 DUT 設備已正確設定網路連線
  • 某些 DUT 專用指令（如 diag）需要特定的環境變數
  • 檔案傳輸時請確保目標磁碟有足夠空間
  • 建議在執行重要操作前先測試連線

📞 技術支援

如遇到問題，請檢查：

  • 程式輸出的錯誤訊息
  • DUT 設備的狀態指示燈
  • 網路連線狀態
  • 防火牆設定
